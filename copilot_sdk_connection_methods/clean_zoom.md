# Office 增益集快取清理指南 (Troubleshooting)

本文件說明何時及如何清理 Office 增益集的本機快取，以確保開發中的程式碼變動能正常生效。

---

## 1. 清理路徑
請在檔案總管中導航至以下路徑並清空其所有內容：

`C:\Users\benit\AppData\Local\Microsoft\Office\16.0\Wef`

---

## 2. 執行清理的時機 (When to Clean)
當遇到以下情況時，強烈建議執行手動清理：

1. **修改 HTML 結構**：例如新增或修改了 `taskpane.html` 的 ID 或元素結構。
2. **Manifest 變動**：修改了 `manifest.xml` 中的資訊（如 標題、圖示、Ribbon 按鈕或權限）。
3. **熱更新 (HMR) 失效**：明明修改了 CSS 或 JS/TS 代碼，但畫面重整後依然顯示舊版。
4. **開發疑難雜症**：當程式碼檢查正確但執行結果怪異，或是 `localStorage` 資料無法清除時。

---

## 3. 正確執行步驟
1. **完全關閉** 所有執行中的 Word 程式。
2. 開啟檔案總管，進入上述 **Wef 路徑** 並 **刪除所有子資料夾**。
3. 返回開發終端機，重啟 `npm run dev` 並 **重新 Side-load (重啟增益集)**。

> [!TIP]
> 定時清理可避免舊資料干擾開發設定，確保開發環境始終載入最準確的程式碼。
